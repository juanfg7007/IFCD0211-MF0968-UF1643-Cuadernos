UF1643. UD4. Actividad 1. Caso práctico: Modelos de auditoría y control de sistemas de información


Enunciado (Descripción)

Revisa los modelos de auditoría y control de sistemas de información en los casos que se enuncian a continuación.


Preguntas / Actividades a realizar 

Verifica el mecanismo para auditar los controles de aplicación en los siguientes casos:

Actividad 1: Instala el software gratuito WireShark (www.wireshark.org) en un ordenador personal para la auditoría a los controles de aplicación, incluyendo: flujo de las transacciones a través del sistema, modelo de estudio de riesgos para analizar los controles de las aplicaciones, observar y probar los procedimientos realizados por los usuarios, prueba de integridad de los datos en los sistemas de procesamiento de transacciones en línea, sistematización de aplicación de pruebas y auditoría continua en línea mediante diferentes técnicas. Usa un protocolo de transferencia de archivos (FTP), analízalo con el software de auditoría instalado y envía la imagen de la captura de la pantalla.

Actividad 2: Utiliza el software Belarc (www.berlarc.com) para auditar el hardware y el sistema operativo de los equipos utilizados para instalar la plataforma bajo Moodle con la finalidad de dar soporte técnico remoto. Envía un archivo PDF del reporte, comenta los aspectos destacados y dos de las amenazas detectadas.

Actividad 3: Muestra los registros de estadísticas de TU acceso como administrador de la plataforma académica de una universidad bajo Moodle. Envía la imagen de la captura de la pantalla y su localizador universal de recursos en la web o Universal ResourceLocator (URL).

Informe analítico sobre el uso de Wireshark para auditar controles de aplicación en el contexto de FTP

En el contexto de los modelos de auditoría y control de sistemas de información, Wireshark se destaca como una herramienta esencial para la auditoría de los controles de aplicación. Este informe analiza el uso de Wireshark para auditar el protocolo de transferencia de archivos (FTP), abarcando el flujo de transacciones, análisis de riesgos, observación de procedimientos de usuario, pruebas de integridad de datos y auditoría continua.

Wireshark es un analizador de protocolos de red de código abierto ampliamente utilizado para capturar y analizar el tráfico de red en tiempo real. Permite a los auditores inspeccionar los datos a nivel de paquete, identificar problemas y verificar la correcta implementación de los controles de aplicación.


Funciones clave

  • Captura de tráfico en tiempo real.
  • Análisis detallado de protocolos.
  • Filtros de visualización y captura.
  • Seguimiento de flujos de datos.
  • Soporte para múltiples protocolos de red.


Flujo de transacciones a través del sistema

Wireshark permite capturar y analizar el flujo de transacciones a través del sistema, proporcionando una visión detallada de las comunicaciones entre los clientes FTP y el servidor FTP. Los pasos para capturar y analizar el tráfico FTP son:

  • Captura de tráfico
    • Iniciar Wireshark y seleccionar la interfaz de red apropiada para capturar el tráfico.
    • Aplicar un filtro de captura para centrarse en el tráfico FTP: TCP PORT 21.
    • Iniciar la captura de tráfico.

  • Análisis del flujo de transacciones
    • Utilizar el filtro de visualización FTP para ver solo los paquetes relacionados con FTP.
    • Inspeccionar las secuencias de comandos FTP (comandos y respuestas) para verificar el flujo de transacciones.
    • Identificar las fases de la transacción: conexión, autenticación, transferencia de archivos y cierre de sesión.


Modelo de estudio de riesgos para analizar los controles de las aplicaciones

El análisis de riesgos en el contexto de FTP implica identificar posibles vulnerabilidades y evaluar el impacto y la probabilidad de amenazas. Wireshark ayuda a identificar estas vulnerabilidades observando el tráfico de red.

  • Factores de riesgo comunes en FTP
    • Transmisión de datos en texto plano: FTP transmite datos, incluidas las credenciales de autenticación, en texto plano, lo que lo hace vulnerable a la intercepción.
    • Accesos no autorizados: análisis de intentos de acceso no autorizados y fuerza bruta.

  • Análisis de riesgos con Wireshark
    • Identificación de credenciales en texto plano:
      • Capturar el tráfico de autenticación FTP y buscar comandos USER y PASS.
      • Verificar si las credenciales se transmiten en texto plano.
    • Detección de accesos no autorizados:
      • Monitorizar intentos fallidos de inicio de sesión.
      • Identificar patrones de fuerza bruta observando múltiples intentos de autenticación fallidos en un corto período. 


Observación y prueba de procedimientos realizados por los usuarios

Wireshark puede usarse para observar los procedimientos realizados por los usuarios y verificar si se siguen los procedimientos de seguridad adecuados.

  • Procedimientos de seguridad comunes en FTP
    • Uso de comandos seguros.
    • Restricción de acceso a ciertos directorios.
    • Monitoreo de la transferencia de archivos.

  • Pruebas con Wireshark
    • Monitorización de comandos FTP
      • Capturar y analizar los comandos FTP utilizados por los usuarios.
      • Verificar que se utilizan comandos seguros y que se siguen las políticas de seguridad.
    • Acceso a directorios
      • Observar los comandos CWD y LIST para verificar los accesos a directorios.
      • Asegurarse de que los usuarios solo acceden a los directorios permitidos.


Prueba de integridad de los datos en los sistemas de procesamiento de transacciones en línea

La integridad de los datos es crucial en las transacciones FTP, asegurando que los archivos no se corrompan durante la transferencia.

  • Pruebas de integridad de datos con Wireshark
    • Captura de transferencias de archivos
      • Capturar el tráfico durante la transferencia de archivos.
      • Analizar los paquetes de datos y verificar la secuencia de datos.
    • Verificación de integridad
      • Comparar los datos capturados con los datos originales.
      • Utilizar checksums o hash para verificar que los archivos no se corrompieron durante la transferencia.


Sistematización de aplicación de pruebas y auditoría continua en línea mediante diferentes técnicas

Wireshark permite la automatización y la sistematización de las pruebas y auditorías continuas a través de la configuración de filtros y alertas. Técnicas de auditoría continua:

  • Configuración de filtros de captura:
    • Crear filtros específicos para capturar tráfico relacionado con eventos críticos, como autenticaciones y transferencias de archivos.
    • Guardar filtros para reutilización en auditorías futuras.
  • Alertas y notificaciones:
    • Configurar Wireshark para generar alertas cuando se detecten actividades sospechosas, como múltiples intentos fallidos de inicio de sesión.
    • Integrar Wireshark con sistemas de gestión de eventos e información de seguridad (SIEM) para una respuesta en tiempo real.


Caso de uso: ejemplo de auditoría de un protocolo de transferencia de archivos (FTP)

Objetivo: auditar el uso del protocolo FTP en una empresa para garantizar la seguridad de las transferencias de archivos y la integridad de los datos.

Acciones:

  • Capturar el tráfico FTP en un periodo de operación típico.
  • Analizar la captura para identificar posibles vulnerabilidades, como la transmisión de credenciales en texto plano y accesos no autorizados.
  • Verificar la integridad de los archivos transferidos comparando los datos capturados con los originales.
  • Configurar alertas para detectar actividades sospechosas en tiempo real.

Resultados esperados:

  • Identificación de vulnerabilidades en la transmisión de datos en texto plano.
  • Detección de patrones de acceso no autorizado.
  • Verificación de la integridad de los datos transferidos.
  • Implementación de alertas para una auditoría continua y respuesta rápida a incidentes.
     

Informe analítico sobre el uso de Belarc para auditar el hardware y sistema operativo de los equipos utilizados para instalar Moodle

En el contexto de la administración y gestión de sistemas de información, la auditoría del hardware y del sistema operativo de los equipos que soportan la plataforma Moodle es crucial para garantizar su eficiencia, seguridad y fiabilidad. Belarc Advisor es una herramienta utilizada para este propósito, proporcionando una auditoría detallada de los sistemas. Este informe analiza el uso de Belarc, sus aspectos destacados y las posibles amenazas detectadas.

Belarc Advisor es una herramienta de auditoría y gestión de activos que proporciona un análisis detallado del hardware, software y configuración de seguridad de los sistemas informáticos. Belarc genera un perfil completo y detallado del sistema, que incluye información sobre el hardware, el software instalado, las actualizaciones de seguridad faltantes, y las políticas de seguridad.


Funciones clave

  • Inventario de hardware: detalla los componentes de hardware, como CPU, RAM, discos duros, y dispositivos conectados.
  • Inventario de software: enumera todo el software instalado, incluidas las versiones y las licencias.
  • Actualizaciones de seguridad: identifica las actualizaciones de seguridad que faltan en el sistema operativo.
  • Configuración de red: proporciona información sobre la configuración de red, incluidas las direcciones IP y MAC.
  • Políticas de seguridad: analiza y reporta sobre las políticas de seguridad implementadas en el sistema. 


Aspectos destacados

  • Inventario completo de hardware y software: Belarc proporciona un inventario detallado de todos los componentes de hardware y software del sistema, incluyendo detalles como el número de serie, modelo, y versión del software. Esto es crucial para el soporte técnico remoto y para asegurar la compatibilidad del sistema con Moodle.

  • Identificación de actualizaciones de seguridad faltantes: Belarc verifica las actualizaciones de seguridad del sistema operativo y enumera las que faltan. Esto permite a los administradores asegurar que los sistemas estén actualizados y protegidos contra vulnerabilidades conocidas.

  • Análisis de políticas de seguridad: Belarc evalúa las políticas de seguridad implementadas en el sistema, proporcionando información sobre configuraciones de seguridad críticas, como las políticas de contraseñas y los permisos de usuario.

  • Facilidad de uso: la interfaz de Belarc es intuitiva y fácil de usar, lo que permite a los administradores ejecutar auditorías de manera rápida y eficiente sin necesidad de una formación extensa.


Amenazas detectadas

  • Actualizaciones de seguridad faltantes: si Belarc detecta que faltan actualizaciones de seguridad críticas, esto representa una amenaza significativa, ya que las vulnerabilidades no corregidas pueden ser explotadas por atacantes.

  • Configuraciones de seguridad débiles: Belarc puede identificar configuraciones de seguridad débiles, como políticas de contraseñas inadecuadas o permisos de usuario excesivos. Estas configuraciones pueden comprometer la seguridad del sistema y facilitar accesos no autorizados.

  • Software no autorizado: la detección de software no autorizado o potencialmente peligroso en el sistema puede indicar un riesgo de seguridad. Es esencial que los sistemas solo tengan instalados el software necesario y autorizado para reducir la superficie de ataque.

  • Falta de documentación y licencias: la falta de licencias válidas para el software instalado puede representar un riesgo legal y de cumplimiento. Belarc ayuda a identificar estas deficiencias para que puedan ser corregidas.
