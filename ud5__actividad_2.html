UF1643. UD5. Actividad 2. Ejercicio teórico-práctico: Identificar y solucionar incidencias en el sistema de gestión de información


Enunciado (Descripción)

Haz una propuesta para auditar los intentos de acceso fallido a una plataforma académica bajo Moodle que solo permite acceso manual y no admite invitados.


Preguntas / Actividades a realizar 

Actividad 1: Selecciona una muestra de los accesos fallidos a la plataforma. Envía la imagen de la captura de la pantalla afectada y su correspondiente URL.

Actividad 2: Indica un mecanismo que permita acelerar la velocidad de respuesta. Envía la imagen de la captura de la pantalla afectada y su correspondiente URL.

      Mecanismos para acelerar la velocidad de respuesta

      Para acelerar la velocidad de respuesta del sistema y mejorar el rendimiento general de la plataforma Moodle, se pueden implementar varios mecanismos y configuraciones


      Optimización del caché de aplicaciones y Bases de Datos

      • Habilitación de caché de aplicaciones
        • Utilizar sistemas de caché como Memcached o Redis para almacenar en memoria los datos más solicitados, reduciendo el tiempo de acceso y carga.

      • Configuración de caché de Bases de Datos
        • Implementar un sistema de caché para consultas frecuentes a la base de datos, mejorando el tiempo de respuesta para operaciones repetitivas.
        • Utilizar herramientas específicas del gestor de bases de datos, como el caché de consultas en MySQL.


      Ajustes de rendimiento de PHP

      • Configuración de Opcache
        • Habilitar y configurar Opcache para almacenar en caché los scripts de PHP compilados, reduciendo el tiempo de carga de las páginas.
        • Ajustar parámetros como opcache.memory_consumption y opcache.max_accelerated_files para optimizar el rendimiento.

      • Ajustes de PHP-FPM
        • Utilizar PHP-FPM (FastCGI Process Manager) para gestionar eficientemente los procesos PHP, mejorando el rendimiento bajo cargas altas.
        • Configurar los parámetros de pm.max_children y pm.start_servers según los recursos del servidor y el tráfico esperado.


      Mejora de la configuración del servidor web

      • Optimización de Nginx o Apache
        • Ajustar los parámetros de configuración del servidor web para mejorar la gestión de conexiones y el tiempo de respuesta.
        • Para Nginx, ajustar parámetros como worker_processes, worker_connections y client_max_body_size.
        • Para Apache, ajustar parámetros como MaxRequestWorkers, KeepAlive y KeepAliveTimeout.

      • Uso de HTTP/2
        • Habilitar HTTP/2 para mejorar la eficiencia de la transferencia de datos y reducir la latencia en la comunicación entre el servidor y el cliente.
        • Asegurarse de que el servidor web y los navegadores de los usuarios soporten HTTP/2.


      Implementación de balanceo de carga

      • Distribución de tráfico
        • Utilizar un balanceador de carga para distribuir el tráfico entre múltiples servidores, reduciendo la carga individual y mejorando la disponibilidad.
        • Configurar el balanceador para manejar adecuadamente el tráfico HTTPS y mantener las sesiones de los usuarios.

      • Escalado horizontal
        • Implementar escalado horizontal añadiendo más servidores para manejar el aumento del tráfico, asegurando que la plataforma pueda crecer con la demanda.


      Monitoreo y análisis del rendimiento

      • Implementación de herramientas de monitoreo
        • Utilizar herramientas como Nagios, Zabbix o New Relic para monitorizar el rendimiento del servidor y la aplicación en tiempo real.
        • Configurar alertas para identificar y responder rápidamente a problemas de rendimiento.

      • Análisis de cuellos de botella
        • Analizar los datos de monitoreo para identificar cuellos de botella en el rendimiento, como consultas lentas a la base de datos o alta utilización de CPU.
        • Implementar soluciones específicas para cada cuello de botella identificado.
